## 项目背景

财务报表解读依赖人工分析，效率低、一致性差。核心挑战是将非结构化的分析诉求转化为可靠的数据查询，并结合大模型生成可信的财务洞察。

## 角色与职责

- 核心架构设计者与技术负责人
- 负责 OLAP 数据层设计与 LLM 接入方案
- 设计 Prompt 工程与输出校验机制

## 技术挑战

- LLM 生成的 SQL 准确率难以保证，容易产生幻觉
- 财务数据敏感，需要严格的权限控制与审计
- OLAP 查询延迟需控制在秒级，才能满足交互体验

## 解决方案

- 架构：StarRocks（OLAP 层）+ LLM（解读层）+ 业务规则校验层
- 设计"查询 → 校验 → 解读"三阶段 Pipeline，LLM 生成 SQL 后经规则引擎校验再执行
- 建立财务指标 Schema 描述文件，作为 LLM Context 注入，减少幻觉

## 成果

- 财务报表解读耗时从小时级降至分钟级
- 支撑 CFO 级别周报、月报自动解读
- SQL 生成准确率 > 90%（经规则校验）
